# Flashcards — Software Engineering Foundations

Covers: Final Review slide 7. Source: Lessons 13–14 framing material.

---

**Q:** What is software engineering, in the framing used in this course?

**A:** Software engineering is **programming integrated over time** — it encompasses not just the act of writing code, but **all of the tools and processes an organization uses to build and maintain that code over time**.

---

**Q:** Finish the sentence: Software engineering is more than coding because it must keep code _______ over its life cycle.

**A:** **sustainable** — able to react to necessary change from conception through introduction, maintenance, and deprecation.

---

**Q:** Name the four life-cycle phases mentioned in the definition of "sustainable" software.

**A:** **Conception → Introduction → Maintenance → Deprecation.**

---

**Q:** What is the difference between programming and software engineering?

**A:** **Programming** is the act of writing code (a one-time creation activity). **Software engineering** is programming **integrated over time** — it adds the discipline, tools, and processes needed to keep that code valuable, correct, and changeable for years.

---

**Q:** (MCQ — select all that apply) Which of the following are concerns of software engineering, but not of programming alone?

  A. Choosing a variable name
  B. Keeping code valuable over the long term
  C. Designing for predictable change
  D. Hitting Enter at the end of a line
  E. Making the discipline itself more rigorous

**A:** **B, C, E.** A and D are pure coding concerns; software engineering layers organizational discipline and longevity on top.

---

**Q:** What three guiding questions does Lesson 30 frame around the definition of software engineering?

**A:**
1. What practices keep code valuable over the long term?
2. How can engineers make a codebase more sustainable and the discipline itself more rigorous?
3. What practices make code able to react to necessary change throughout its life cycle?
